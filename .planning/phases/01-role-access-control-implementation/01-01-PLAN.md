| Boundary          | Description                                                                               |
| ----------------- | ----------------------------------------------------------------------------------------- |
| client→middleware | Unauthenticated request reaches authMiddleware; resolved user context flows to all guards |
| page→can()        | Client render bodies and server loaders call can() on the resolved user                   |

## STRIDE Threat Register

| Threat ID | Category               | Component                       | Severity | Disposition | Mitigation Plan                                                                                                                                                                 |
| --------- | ---------------------- | ------------------------------- | -------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| T-01-01   | Elevation of Privilege | can() null handling             | low      | mitigate    | `can(null, action)` returns false; guards keep explicit `NoUserContextError` for null before calling can()                                                                      |
| T-01-02   | Elevation of Privilege | can(user,'admin') using isStaff | high     | mitigate    | Use `user.role === 'staff'` (effective role), NOT `isStaff`, so staff previewing as learner (role flipped to 'learner') lose admin — preserves auth.ts effective-role semantics |
| T-01-03   | Elevation of Privilege | accessChallenges for staff      | medium   | mitigate    | Preserve current behavior: `accessChallenges` = lite                                                                                                                            | pro only; staff denied unless they hold a sub (matches CHALLENGE_ACCESS map) |
| T-01-04   | Tampering              | Role enum narrower than DB      | medium   | mitigate    | Wave 2 widens AdminUserEdit/AdminCreateUser `z.enum(['learner','staff'])` to `z.enum(USER_ROLES)` so valid lite/pro roles are accepted                                          |
| T-01-SC   | Tampering              | npm/pip/cargo installs          | high     | mitigate    | No package installs in this phase (pure refactor); slopcheck N/A                                                                                                                |
